\doxysubsection{Detailed Description}
This controller uses routes defined on actions with a route defined at the controller-\/level. 

Routes can be defined at the controller level by placing the \mbox{[}Route\mbox{]} attribute on the controller class. Any route specified in this way should include \{action\} as a parameter, so that it\textquotesingle{}s clear which action is to be invoked. If a request matches multiple actions, it will return an HTTP 404.

In this example, the \mbox{[}Route\mbox{]} attribute on the controller class will match a URL pattern like \textquotesingle{}/\+Students/\+Edit/5\textquotesingle{}. The \textquotesingle{}Students\textquotesingle{} prefix is prepended to the template. The \{id\+:int\} parameter forces the required id parameter to have an integer value. This route can only match the Edit, Details, and Delete actions on this controller.

The \mbox{[}Route\mbox{]} attribute defined on the controller class also provides a Name so that it can used to generate URLs via the Html.\+Route\+Link method. In attribute routing, actions and controllers generally are configured to match multiple URL patterns using the Name property allows you to have control over which routes are used to generate links.

Routes defined on actions override any routes that are defined on the controller. For instance, the Index action has two \mbox{[}Route\mbox{]} attribute which will match \textquotesingle{}/\+Students\textquotesingle{} and \textquotesingle{}/\+Students/\+Index\textquotesingle{}. These routes replace the route defined at the controller level. \textquotesingle{}/\+Students/\+Index/5\textquotesingle{} will not match this action. Note that the \mbox{[}Route\+Prefix\mbox{]} defined at the controller level still applies.
