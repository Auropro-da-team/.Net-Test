Detailed Description

This controller demonstrates default ordering of attribute routes. In general, as per the routing guidelines, more specific routes should be registered before generic ones. Attribute routing by default tries to do the best it can to order the routes according to it.

For the following controller, the routes are ordered like the following:

  1. api/categories/all
  2. api/categories/{id:int}
  3. api/categories/{name}
