Detailed Description

This controller demonstrates mixing conventional and attribute routing together in a single controller. Here 'GetOrdersOfCustomer' is the only action which uses attribute routing, where as the remaining actions are accessible through conventional route.

NOTE:

  • Conventional routes can never access actions/controllers which are decorated with route attributes.
